Setup & Installation

Get started with Vibe UI templating system.

Getting Started

Vibe UI is a high-performance, utility-first UI templating system designed for building stunning, themeable admin dashboards and web applications. Built with Vanilla JS and Tailwind CSS.

1. Installation

Clone the repository directly into your project's root directory.

git clone https://github.com/your-repo/vibe-templating.git .

2. Customization

Edit themes.json in the root directory to set your desired initial state. Modifying this file will change the starting theme, accent color, and layout for all new users.

Configuration & Theming

Vibe UI uses a centralized configuration system powered by themes.json located in the root directory. This file defines the initial defaults for the application.

Key Description Available Values
color-theme Sets the default theme mode on first load. "light", "dark"
light-palette Selection of color schemes for Light Mode. "Pure White", "Soft Slate", "Modern Gray", "Oceanic Light", "Professional Cream"
dark-palette Selection of color schemes for Dark Mode. "Classic Slate", "Midnight Onyx", "Charcoal Elegance", "Oceanic Deep", "Antigravity IDE", "VS Code Modern"
accent-color The primary brand/accent color in RGB format. Any RGB string (e.g., "79, 70, 229")
bg-preset The background and surface color strategy. "Neutral", "Solid", "Matching"
canvas-style The outer layout container style. "Full", "Boxed", "Soft", "Floating"
zoom-level Controls the base font size and UI scaling. "Compact", "Standard", "Zoom"

Technology Stack

Core Logic

HTML5, Vanilla JavaScript (ES6+). Zero heavy frameworks required.

Styling Engine

Tailwind CSS via custom Plugin Architecture utilizing @apply.

Floating Elements

Powered by Floating UI for robust dropdowns and tooltips.

Typography & Icons

Inter Font via Google Fonts, and Boxicons for vector iconography.

Method 2: Manual Integration

Copy the dist/ folder into your project's root or public directory. Then, link the CSS and JS files in your HTML.

<!-- Inside <head> -->
<script src="dist/js/vendor/tailwindcss.js"></script>
<link href="dist/css/boxicons.min.css" rel="stylesheet">
<script src="dist/js/tailwind-config.js"></script>
<link rel="stylesheet" href="dist/css/theme.css">
<link rel="stylesheet" href="dist/css/layout.css">
<link rel="stylesheet" href="dist/css/components.css">

<!-- Just before </body> -->
<script src="dist/js/vendor/floating-ui.core.min.js"></script>
<script src="dist/js/vendor/floating-ui.dom.min.js"></script>
<script src="dist/js/config.js"></script>
<script src="dist/js/menu.js"></script>
<script src="dist/js/app.js"></script>

Method 2: Manual Integration

1. Base Layout (app/Views/layouts/main.php)

Extract the outer shell of page-blank.html. This includes the HTML head, loader, sidebar overlay, and the main layout grid.

<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Include meta tags and CSS references here -->
    <title><?= $this->renderSection('title') ?> - Vibe UI Kit</title>
</head>
<body class="bg-body text-slate-900 dark:text-slate-100 font-sans transition-colors duration-300">
    <div id="app-loader"><div class="loader-spinner"></div></div>

    <div id="app-layout" class="flex w-full h-screen overflow-hidden transition-all duration-500 bg-body">
        <div id="sidebar-overlay" class="fixed inset-0 bg-black/50 backdrop-blur-sm z-40 hidden transition-opacity duration-300 opacity-0"></div>

        <!-- Include Sidebar Partial -->
        <?= $this->include('partials/sidebar') ?>

        <div id="content-wrapper" class="flex flex-col flex-1 min-w-0 overflow-hidden">
            <!-- Include Header Partial -->
            <?= $this->include('partials/header') ?>

            <main id="main-content" class="flex-1 overflow-y-auto w-full space-y-8 transition-all duration-500">
                <div id="main-canvas" class="w-full space-y-8 transition-all duration-500 p-6">
                    <!-- Dynamic Page Content -->
                    <?= $this->renderSection('content') ?>
                </div>
            </main>
        </div>
    </div>

    <!-- Vendor Scripts -->
    <?= $this->renderSection('scripts') ?>
</body>
</html>
2. Partials (app/Views/partials/)

Copy the <aside id="sidebar"> into sidebar.php and the <header id="app-header"> into header.php. This keeps your main layout clean and allows you to update navigation globally.

3. Content Page (app/Views/pages/blank.php)

Now, you can create views that extend your base layout and inject specific content inside the #main-canvas.

<?= $this->extend('layouts/main') ?>

<?= $this->section('title') ?>
Utility / Blank Page
<?= $this->endSection() ?>

<?= $this->section('content') ?>
    <div class="mb-8">
        <h2 class="text-3xl font-extrabold mb-2">Blank Page</h2>
        <p class="text-muted font-medium">Start your new project from here.</p>
    </div>

    <div class="h-[60vh] border-2 border-dashed border-slate-200 dark:border-slate-800 rounded-[2rem] flex flex-col items-center justify-center text-center p-8">
        <div class="w-16 h-16 rounded-2xl bg-alt flex items-center justify-center text-muted mb-6">
            <i class="bx bx-plus text-3xl"></i>
        </div>
        <h4 class="text-xl font-black mb-2 italic text-slate-400">Your content goes here...</h4>
        <p class="text-sm text-slate-400 max-w-sm font-medium">This is a clean container to help you start building unique sections for your application.</p>
    </div>
<?= $this->endSection() ?>
