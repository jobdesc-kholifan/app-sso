AI Vibe Coding

Leverage AI to seamlessly generate and integrate your layouts.

What is Vibe Coding?

"Vibe Coding" is our methodology for integrating the Vibe UI template using Large Language Models (like Claude, ChatGPT, or GitHub Copilot). Instead of manually copying and pasting HTML, you can use these tailored prompts to have the AI generate your layouts automatically.

Phase 0: Fetching the Template

Before using the AI prompts, you must first clone the Vibe UI repository into your project root under a specific folder name (docs-template).

git clone https://github.com/your-repo/vibe-templating.git docs-template

Phase 1: Initializing Setup

Use this universal prompt first to initialize your project, set up the static assets, and create a root landing page. The AI will automatically adapt to whatever framework you are using.

I have a frontend/backend project and want to initialize the "Vibe UI" templating system. I have cloned the template into a folder named `docs-template` at the root of my project.

Please execute the following steps precisely:

1. **Check Framework**: First, detect the framework or environment I am currently using in this project (e.g., CodeIgniter 4, Laravel, Express, etc.). If you cannot detect any framework, abort this process immediately.
2. **Copy Assets**: Based on the detected framework, determine the correct public target directory (e.g., `public/`, `htdocs/`, or the root directory). Automatically copy the entire `docs-template/dist/` folder into that target directory.
3. **Setup Landing Route**: Create a root route (`/`) using the routing system of the detected framework.
4. **Create View**: Generate a view file for that `/` route. Use the layout structure and HTML content from `docs-template/pages/page-landing.html` as the exact reference for this view. 
   - CRITICAL: Ensure all CSS and JS asset paths inside the view are linked correctly using the framework's native asset/URL helper (e.g., `base_url('dist/css/theme.css')` in CI4 or `asset('dist/css/theme.css')` in Laravel). DO NOT use relative paths like `../dist/` or `dist/` as they will cause 404 errors.
   - For the "Get Started" and "Documentation" buttons, modify their `href` attributes to point to the raw HTML files inside the `docs-template` folder using the framework's native URL helper (e.g., `base_url('docs-template/pages/docs-setup.html')` and `base_url('docs-template/dashboards/index.html')`).

Do not just provide the code. Please directly execute these changes and create/modify the necessary files in my project.

Phase 2: Framework Integration (CodeIgniter 4)

After initializing, use this prompt to instruct your AI assistant to generate the specific layout structure for CodeIgniter 4.

I have an HTML UI template called "Vibe UI" and I need to integrate it into my CodeIgniter 4 (CI4) project. I have cloned the template into a folder named `docs-template` at the root of my project.

First, provide instructions to copy the entire `docs-template/dist/` folder directly into the `public/` directory of my CI4 project.

Next, please help me convert the boilerplate layout (`docs-template/pages/page-blank.html`) into a CI4 view layout structure. Follow these exact steps:

1. Create a base layout file at `app/Views/layouts/main.php`. It should contain the HTML <head>, the `div#app-loader`, and the main wrapper `div#app-layout`.
2. Inside the base layout, use `<?= $this->renderSection('title') ?>` for the page title.
3. Extract the `<aside id="sidebar">` into a partial view at `app/Views/partials/sidebar.php` and include it in the main layout using `<?= $this->include('partials/sidebar') ?>`.
4. Extract the `<header id="app-header">` into a partial view at `app/Views/partials/header.php` and include it inside the `div#content-wrapper`.
5. Inside the `main#main-content > div#main-canvas`, place `<?= $this->renderSection('content') ?>` where the dynamic page content should go.
6. Create an example page view (e.g., `app/Views/pages/blank.php`) that extends `layouts/main`, sets the `title` section, and puts the page-specific HTML content inside the `content` section.
7. Ensure all CSS and JS asset paths correctly reference the assets we copied to the public directory using the base_url helper (e.g., `<?= base_url('dist/css/theme.css') ?>`).

Please output the exact code for `main.php`, `sidebar.php`, `header.php`, and `blank.php`.
